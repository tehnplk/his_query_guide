# 🏥 HOSXP Database Schema Guide (รพ. & รพสต.)

**คู่มือและ Data Dictionary ฉบับนักพัฒนา (Developer Guide)**
จัดทำขึ้นเพื่อเป็นแนวทางสำหรับ Developer ในการศึกษาโครงสร้างฐานข้อมูล HOSXP และร่างคำสั่ง SQL Query โดยเน้นแกนกลางความสัมพันธ์ที่:

- 👤 **`hn` (Hospital Number)** = รหัสประจำตัวคนไข้ (ผูกผู้ป่วย 1 คน)
- 🏥 **`vn` (Visit Number)** = รหัสการมารับบริการผู้ป่วยนอก (OPD) 1 ครั้ง
- 🛌 **`an` (Admit Number)** = รหัสการนอนโรงพยาบาลผู้ป่วยใน (IPD) 1 ครั้ง

## 👥 1. ข้อมูลพื้นฐานประชากร ที่อยู่อาศัย ประเภทอาศัย สิทธิรักษา

- **ทะเบียนประชากร (CID)**: `person`
- **ทะเบียนผู้ป่วย (HN)**: `patient`
- **ที่อยู่อาศัย**: `house`, `village` (ตารางรหัสตำบล/อำเภอ/จังหวัด อ้างอิงจาก `thaiaddress`)
- **ประเภทอาศัย (Typearea)**: `house_regist_type_id`
- **สิทธิการรักษา**: `pttype`

## 🕒 2. เส้นทางข้อมูลคนไข้ (Patient Journey Workflow)

ลำดับการตรวจสอบข้อมูลในแต่ละขั้นตอนการมาโรงพยาบาล

| ลำดับ (Step)               | โมดูล (Module)       | **Transaction Table**                          | **Master Table**                         |
| :------------------------- | :------------------- | :--------------------------------------------- | :--------------------------------------- |
| **1. ก่อนมาโรงพยาบาล**     | นัดหมาย/รับส่งต่อ    | `oapp`, `referin`                              | `clinic`, `hospcode`                     |
| **2. มาถึง/ยืนยันสิทธิ**   | เวชระเบียน (Regis)   | `ovst`, `vn_stat`                              | `pttype`                                 |
| **3. ซักประวัติ/Triage**   | คัดกรอง (Screening)  | `opdscreen`                                    | `kskdepartment`                          |
| **4. พบแพทย์/วินิจฉัย**    | ห้องตรวจ (Exam)      | `ovstdiag`                                     | `icd101`, `doctor`                       |
| **5. ฉุกเฉิน/ส่งต่อ**      | ER / REFER Out       | `er_regist`, `referout`                        | `er_pt_type`, `hospcode`                 |
| **6. ติดตามโรคเรื้อรัง**   | คลินิก NCD (FU)      | `chronic`, `chronicfu`                         | `clinic`                                 |
| **7. บริการวัคซีน (EPI)**  | คลินิกเด็กดี/วัคซีน  | `epi_vaccine`, `oapp_vaccine`                  | `vaccine`                                |
| **8. หัตถการ/ผ่าตัด/แล็บ** | หัตถการ/OR (Technic) | `doctor_operation`, **`iptoprt`**, `lab_order` | `icd9cm1`, `lab_items`, `operation_item` |
| **9. สั่งยา/เวชภัณฑ์**     | ห้องยา (Pharmacy)    | `opitemrece`                                   | `drugitems`, `drugusage`                 |
| **10. ชำระเงิน/รายได้**    | การเงิน (Finance)    | `rcpt_print`                                   | `income`                                 |
| **11. ผู้ป่วยใน (IPD)**    | หอผู้ป่วย (Ward)     | `ipt`, `iptdiag`, `an_stat`                    | `ward`, `spclty`                         |
| **12. จำหน่ายคนไข้**       | จำหน่าย (Discharge)  | `ovst`, `ipt` (อัปเดตสถานะกลับ)                | `ovstist`, `ovstost`, `dchtype`          |

### 2.1 ก่อนมาโรงพยาบาล (Pre-Hospital)

- **ตารางที่เกี่ยวข้อง**: `oapp` (การนัดหมาย), `referin` (รับส่งต่อ)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `oapp`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**
  - `referin`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.2 มาถึง/ยืนยันสิทธิ (Registration)

- **ตารางที่เกี่ยวข้อง**: `ovst` (การเปิด Visit), `vn_stat` (สรุปสถิติผู้ป่วยนอก)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `ovst`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**
  - `vn_stat`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.3 ซักประวัติ/Triage (Screening)

- **ตารางที่เกี่ยวข้อง**: `opdscreen` (จุดคัดกรอง วัดส่วนสูง/น้ำหนัก/ความดัน)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `opdscreen`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.4 พบแพทย์/วินิจฉัย (Examination)

- **ตารางที่เกี่ยวข้อง**: `ovstdiag` (รายการวินิจฉัย)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `ovstdiag`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`** (รหัสโรคใช้ `icd10`)

### 2.5 ฉุกเฉิน/ส่งต่อ (ER / REFER Out)

- **ตารางที่เกี่ยวข้อง**: `er_regist` (ทะเบียน ER), `referout` (ส่งต่อ รพ.อื่น)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `er_regist`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**
  - `referout`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.6 ติดตามโรคเรื้อรัง (NCD Follow-up)

- **ตารางที่เกี่ยวข้อง**: `chronic` (ทะเบียนผู้ป่วย NCD), `chronicfu` (การนัด/ติดตาม)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `chronic`: อ้างอิงคนไข้ด้วย **`hn`** (เป็นทะเบียนประจำตัวโรค)
  - `chronicfu`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.7 บริการวัคซีน (EPI)

- **ตารางที่เกี่ยวข้อง**: `epi_vaccine` (บันทึกวัคซีน), `oapp_vaccine` (นัดวัคซีน)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `epi_vaccine`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.8 หัตถการ/ผ่าตัด/แล็บ (Technic/OR/Lab)

- **ตารางที่เกี่ยวข้อง**: `doctor_operation` (หัตถการ OPD), `iptoprt` (ผ่าตัด IPD), `lab_order`
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `doctor_operation`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับ(OPD)ด้วย **`vn`**
  - `iptoprt`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับ(IPD)ด้วย **`an`**
  - `lab_order`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.9 สั่งยา/เวชภัณฑ์ (Pharmacy)

- **ตารางที่เกี่ยวข้อง**: `opitemrece` (รายละเอียดการสั่งยาและค่าใช้จ่าย)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `opitemrece`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`** (และใช้ `an` หากเป็นยาผู้ป่วยใน)

### 2.10 ชำระเงิน/รายได้ (Finance)

- **ตารางที่เกี่ยวข้อง**: `rcpt_print` (การพิมพ์ใบเสร็จ/การชำระเงิน)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `rcpt_print`: อ้างอิงคนไข้ด้วย **`hn`**, อ้างอิงบริการที่รับด้วย **`vn`**

### 2.11 ผู้ป่วยใน (IPD Admission)

- **ตารางที่เกี่ยวข้อง**: `ipt` (ทะเบียน IPD), `iptdiag` (วินิจฉัย IPD), `an_stat` (สถิติ IPD)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - `ipt`, `iptdiag`, `an_stat`: ทรัพยากรทั้งหมดอ้างอิงคนไข้ด้วย **`hn`** และบริการแอดมิทด้วย **`an`** (Admit Number)
- **วินิจฉัยแรกรับ (Provisional Diagnosis):**
  - เมื่อแพทย์สั่ง Admit (จากบันทึก OPD/ER) ระบบจะจัดเก็บรหัสโรคเบื้องต้นไว้ที่ **`ipt.provision_dx_icd`** รวมถึงคำบรรยายโรคไว้ที่ **`ipt.provision_dx`** เพื่อเป็นข้อบ่งชี้ตั้งต้นก่อนส่งผู้ป่วยขึ้นหอผู้ป่วย

### 2.12 จำหน่ายคนไข้ (Discharge)

- **ตารางที่เกี่ยวข้อง**: `ovst` (สถานะ OPD), `ipt` (สถานะ IPD), `an_stat` (สรุปสถิติ IPD)
- **คีย์อ้างอิงคนไข้และบริการ**:
  - กรณี **OPD**: อ้างอิงคนไข้ด้วย **`hn`**, บริการด้วย **`vn`**
  - กรณี **IPD**: อ้างอิงคนไข้ด้วย **`hn`**, บริการด้วย **`an`**
- **สรุปชาร์จ (Final Diagnosis / Discharge Summary):**
  - หลังจากคนไข้จำหน่ายกลับบ้าน (Discharge) แพทย์จะทำการ "สรุปชาร์จ" รหัสโรคและสถิติสุดท้ายจะถูกจัดเก็บเป็น Final Diagnosis ในตาราง **`an_stat`** (เช่น ฟิลด์ `pdx` = โรคหลัก) และ **`iptdiag`** (รวมโรคร่วม/แทรกซ้อนทั้งหมด) ข้อมูลส่วนนี้จะมีความแม่นยำที่สุด นิยมใช้ทำสรุปรายงานสถิติ และนำไปใช้คิดค่า DRG ส่งเบิกกองทุน สปสช.

### 2.13 ค่าน้ำหนักสัมพัทธ์ (adjRW) และดัชนีความรุนแรงของโรค (CMI)

หลังจากผ่านกระบวนการสรุปชาร์จ (Discharge) และส่งข้อมูลประมวลผลจัดกลุ่มโรคร่วม (DRG Grouper) เรียบร้อยแล้ว ระบบจะเก็บค่าน้ำหนักสัมพัทธ์ที่สะท้อนถึงทรัพยากรและความซับซ้อนของผู้ป่วย โดยมีตารางที่เกี่ยวข้องเพื่อใช้ดึงข้อมูล (จากการ Research Database Schema จริง) ดังนี้:

- **adjRW / RW (ระดับรายบุคคล 1 AN):**
  - **กลุ่มตารางหลักในการรักษา**: 
    - `an_stat` : ฟิลด์ `rw` (มีหลังจากแพทย์สรุปชาร์จจบ)
    - `ipt` : ฟิลด์ `adjrw` และ `rw`
    - `ipt_drg_result` : ฟิลด์ `adjrw` และ `rw` (เก็บผลลัพธ์จากการส่งเข้าโปรแกรม Thai DRG Grouper โดยตรง มีรายละเอียดที่สมบูรณ์ที่สุด นิยมใช้อ้างอิงเวลา Query รายงาน)
    - `ovst_drgs` : ฟิลด์ `adjrw` และ `rw` (กรณีที่มีการทำ OPD DRG ในผู้ป่วยนอก)
  - **กลุ่มตารางส่งออกเพื่อเบิกจ่ายกองทุน**: `rep_eclaim_detail`, `co_rep_link` จะมีบันทึกค่า `adjrw2`, `adjrw_n`, `adjrw_nhso`, `rw` เพื่อไว้ตรวจสอบก่อนส่ง Claim (ชดเชยค่าใช้จ่ายไปยัง สปสช. / กรมบัญชีกลาง)

- **CMI (Case Mix Index - ระดับภาพรวมโรงพยาบาลหรือแผนก):** 
  - **ไม่มีฟิลด์หรือคอลัมน์ `cmi` แยกเก็บในตารางโดยตรง** เนื่องจาก CMI เป็น Aggregate Value (ค่าเฉลี่ยสถิติ)
  - **วิธีดึงข้อมูล**: ต้องเขียน Query คำนวณหาภาพรวมเอง โดยอิงจากข้อมูลล่าสุดตาราง `ipt_drg_result` ด้วยสูตร: `SUM(adjrw) / COUNT(an)`
  - **ประโยชน์**: ผู้บริหารมักใช้วิเคราะห์ประเมินศักยภาพสถานพยาบาล หรือใช้ในการจัดสรรงบประมาณ (หาก CMI > 1 ส่วนใหญ่รักษาโรคที่มีความซับซ้อนสูง)

## 🛏️ 3. ข้อมูลทรัพยากรเตียงและหอผู้ป่วย (Bed Resources & Ward Management)

ส่วนบริหารจัดการพื้นที่รองรับผู้ป่วยใน (IPD) เพื่อเช็คความหนาแน่นและอัตราการครองเตียง

- **ตารางทรัพยากร (Master Tables)**:
  - `ward`: รายการหอผู้ป่วย/ตึก ทั้งหมดในโรงพยาบาล
  - `bedno`: ทะเบียนเตียงรายตัว (ประกอบไปด้วยโครงสร้าง `ward` + `bedno`)
  - `bedtype`: ประเภทของเตียง (เช่นสามัญ, ห้องพิเศษ, ICU)
- **ตารางประวัติความเคลื่อนไหว (Transaction Tables)**:
  - `iptbedmove`: ประวัติการย้ายเตียงหรือการย้ายวอร์ด (เชื่อมโยงแบบ Time-Series)
  - `ipt`: เก็บสถานะเตียงล่าสุดที่นอนอยู่ ณ ปัจจุบัน
- **คีย์อ้างอิงคนไข้และห้อง**:
  - ผู้ป่วยผูกมัดกับการนอนด้วย **`an`**
  - สามารถใช้ **`an`** ไป JOIN ดูประวัติการตระเวนย้ายเตียงได้ใน `iptbedmove`

## 💻 4. ตัวอย่างการเขียน Query สำหรับ Developer (SQL Guidelines)

หลักการพื้นฐานในการดึงข้อมูล คือการเริ่มจากคนไข้ (`patient`) ไปหาการรับบริการ (`ovst`) และต่อยอดไปหาประวัติเฉพาะจุด เช่น คัดกรอง (`opdscreen`) หรือ จ่ายยา (`opitemrece`)

### 📌 Pattern 1: การดึงประวัติการมารับบริการผู้ป่วยนอก (OPD)

```sql
SELECT
    p.hn, p.fname, p.lname,
    v.vn, v.vstdate, v.vsttime,
    s.bps, s.bpd, s.bw -- ข้อมูลความดันและน้ำหนัก
FROM patient p
JOIN ovst v ON v.hn = p.hn
LEFT JOIN opdscreen s ON s.vn = v.vn
WHERE v.vstdate = '2026-03-30';
```

### 📌 Pattern 2: การดึงประวัติผู้ป่วยใน (IPD) และ หัตถการ

```sql
SELECT
    p.hn, p.fname, p.lname,
    i.an, i.regdate, i.dchdate,
    op.icd9, op.opdate -- ข้อมูลหัตถการผ่าตัด
FROM patient p
JOIN ipt i ON i.hn = p.hn
LEFT JOIN iptoprt op ON op.an = i.an
WHERE i.regdate BETWEEN '2026-03-01' AND '2026-03-31';
```

### 📌 Pattern 3: การดึงรายชื่อผู้ป่วยที่ครองเตียงอยู่ ณ ปัจจุบัน (Current Admitted)

```sql
SELECT
    p.hn, p.fname, p.lname,
    i.an, i.regdate, i.ward,
    w.name as ward_name,
    i.bedno
FROM ipt i
JOIN patient p ON p.hn = i.hn
LEFT JOIN ward w ON w.ward = i.ward
WHERE i.dchdate IS NULL; -- ยังไม่จำหน่าย = ครองเตียงอยู่
```
